# بوت العقار الذكي 🏗️

بوت يفهم طلبك العقاري بالعربي (عن طريق Groq المجاني) ويسحب لك إعلانات حقيقية
من موقع **عقار (aqar.fm)** — السعر، المساحة، رقم الإعلان، ورقم الجوال لو كان
مكتوب داخل نص الإعلان.

---

## ⚠️ مهم تعرفه قبل النشر

1. **عقار.فم ما عندها API رسمي.** هذا الكود يسحب بيانات من صفحات عامة (مش
   محمية بكلمة سر) — يعني مناسب للاستخدام الشخصي/الخفيف، وما ينصح تستخدمه
   بكثافة عالية (آلاف الطلبات/الساعة) لأن الموقع ممكن يحجبك مؤقتاً.
2. **رقم الجوال** غالباً يكون مكتوب داخل نص الإعلان نفسه (زي "📞 للتواصل:
   05xxxxxxxx")، لكن بعض الإعلانات ما تكتبه — وقتها البوت يعرض رابط الإعلان
   الأصلي بس عشان تتواصل من خلاله.
3. لو غيّرت عقار تصميم موقعها مستقبلاً، ممكن يحتاج كود السحب (`api/search.js`)
   تعديل بسيط — الكود مكتوب بطريقة مرنة (يعتمد على رمز الريال ﷼ ووحدة م²
   بدل أسماء classes) عشان يصمد قدر الإمكان.

---

## 🚀 خطوات النشر على Vercel (مجاني بالكامل)

### 1) احصل على مفتاح Groq مجاني
- روح لـ [console.groq.com](https://console.groq.com)
- سجل دخول بإيميلك (بدون بطاقة بنكية)
- من API Keys أنشئ مفتاح جديد، وخزّنه عندك

### 2) رفع المشروع على GitHub
```bash
cd aqar-bot
git init
git add .
git commit -m "أول نسخة من بوت العقار"
```
أنشئ مستودع جديد على GitHub وارفع الكود له (`git remote add origin ...` ثم
`git push`).

### 3) استضافة على Vercel
- روح [vercel.com](https://vercel.com) وسجل دخول (يفضل بحساب GitHub نفسه)
- اضغط **New Project** واختر المستودع اللي رفعته
- في خطوة **Environment Variables** أضف:
  - `GROQ_API_KEY` = المفتاح اللي حصلت عليه من الخطوة 1
- اضغط **Deploy** وانتظر دقيقة، وراح يعطيك رابط جاهز مباشرة 🎉

### 4) جرّبه
افتح الرابط، واكتب مثلاً:
> أبحث عن قطعة أرض في حي الياسمين بالرياض مساحة 500 متر

---

## 🛠️ تشغيل محلي (لو حاب تجرب على جهازك قبل النشر)

```bash
npm install -g vercel
cd aqar-bot
npm install
vercel dev
```
ثم افتح `http://localhost:3000` وضيف ملف `.env` فيه:
```
GROQ_API_KEY=مفتاحك_هنا
```

> ملاحظة أمان: لا ترسل ملف `.env` أو مفتاح `GROQ_API_KEY` في Git أو للمشاركة العامة. استخدم ملف `.env` محلياً فقط، واحتفظ بالمفتاح سري.

## 🔧 إضافة منصات أخرى (حراج، وصلت...) مستقبلاً

كل منصة لها بنية HTML مختلفة تماماً، فلازم ملف سحب (`scraper`) خاص بها.
الطريقة المقترحة: أنشئ `api/scrapers/haraj.js` بنفس منطق `search.js` (سحب +
تحليل)، وبعدين في `api/search.js` خلي الكود يجرب أكثر من مصدر ويجمع
النتائج مع بعض قبل ما يرجعها للواجهة.
